# Skill 安全与管理工具 V1.0

## 项目简介

**Skill 安全与管理工具** 是一款专为 AI 编码助手（如 Claude Code、Cursor、CodeBuddy 等）设计的 Skill 安全检测与可视化管理平台。它能够帮助开发者：

- **快速检索** 系统中所有 AI 工具的 Skill 目录，并直观展示 Skill 名称、描述、路径、文件数等信息。
- **恶意代码检测** 基于正则规则库（支持自定义）对 Skill 文件进行安全扫描，发现潜在的恶意模式（如命令注入、后门、敏感信息泄露等）。
- **在线批量检测** 支持 GitHub、skills.sh、SkillStore、skill-cn、ClawHub 等平台的 Skill 仓库直接下载并扫描。
- **可视化管理** 将检索到的 Skill 添加到管理列表，支持打开目录、删除目录、发送到安全检测等操作。
- **灵活的规则切换** 提供“全高危扫描”和“恶意高危扫描（精确）”两种模式，用户可自行扩展规则文件。

该工具适用于安全审计人员、AI 应用开发者以及任何希望确保第三方 Skill 安全的用户。

## 主要功能

| 模块           | 功能                                                         |
| -------------- | ------------------------------------------------------------ |
| **Skill 管理** | 拖放 `SKILL.md` 文件自动添加 Skill；自定义目录递归检索；全盘扫描；常见 AI 工具 Skill 路径一键检索。 |
| **安全检测**   | 离线扫描本地文件/目录/ZIP；在线批量检测 GitHub 等平台的 Skill；扫描结果支持右键删除恶意行。 |
| **描述提取**   | 智能解析 `SKILL.md` 中的 YAML Front Matter 和多行描述（支持 `\|`、`>` 等块标量）。 |
| **代理支持**   | HTTP/HTTPS/SOCKS5 代理，可测试连通性。                       |
| **日志与报告** | 调试日志窗口；导出检测报告（含删除操作记录）。               |

## 运行环境要求

### 操作系统
- Windows 7/10/11
- macOS 10.15+
- Linux (Ubuntu 20.04+, Debian 11+, 等)

### Python 版本
- Python 3.8 或更高版本（推荐 3.10+）

### 依赖库
详见 `requirements.txt`：
```
requests>=2.25.0
tkinterdnd2>=0.3.0
PySocks>=1.7.1
urllib3>=1.26.0
```

### 可选规则文件
- `total_rules.py`：全高危操作扫描规则（如系统命令执行、文件篡改等）。
- `Precise_rules.py`：恶意高危操作扫描规则（更贴近实际攻击模式）。

如未提供规则文件，程序仍可运行，但扫描功能将无法使用（仅 Skill 管理功能可用）。

## 安装与运行

1. **克隆仓库**
   ```bash
   git clone https://github.com/Jack-MRJ/Safety-and-Management-Tools-for-Skill.git
   cd skill-security-tool
   ```

2. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

3. **运行程序**
   ```bash
   python skill_manager.py
   ```
   或直接双击 `skill_manager.py`（确保 `.py` 文件关联到 Python）。

4. **首次使用**
   - 程序会自动创建 `managed_skills.json` 用于保存管理列表。
   - 若提示缺少规则文件，请根据需求自行编写或从社区获取。

## 使用指南

### Skill 管理视图（默认启动）

- **自定义扫描目录**：在文本框中输入目录路径（每行一个），点击“检索Skill”即可递归查找所有包含 `SKILL.md` 的目录。
- **检索常见 AI 工具**：自动扫描当前用户目录下的 `.claude/skills`、`.cursor/skills`、`.codex/skills` 等路径（点击 `?` 查看完整路径列表）。
- **全盘扫描**：遍历系统所有磁盘分区（Windows）或 `/home`、`/usr` 等目录，查找 Skill。
- **拖放 `SKILL.md`**：将任意 `SKILL.md` 文件拖放到列表区域，自动将其所在目录添加为一条 Skill 记录。
- **右键菜单**：
  - 打开文件夹（多选）
  - 从列表中移除（仅删除记录，不删除实际文件）
  - 永久删除 Skill 目录
  - 发送到安全检测（将目录添加到离线扫描列表）
  - 从 SKILL.md 文件添加（手动选择文件）

### 安全检测视图

点击工具栏“🔄 切换至安全检测”进入。

- **离线扫描**：
  - 点击“添加文件/目录”或将文件/文件夹拖放到列表区域。
  - 支持 `.zip` 压缩包自动解压扫描。
  - 点击“离线扫描”，结果窗口会显示每个文件的行级命中规则，支持双击跳转到编辑器（VSCode/Sublime/记事本）。
  - 可在结果窗口中右键删除恶意行（永久修改源文件）。
- **在线批量检测**：
  - 在文本框中输入 Skill 的 URL（每行一个），支持 GitHub、skills.sh、SkillStore、skill-cn、ClawHub。
  - 点击“在线检测 (批量)”后，自动下载并扫描，结果以标签页形式展示。
- **代理设置**：支持 HTTP/HTTPS/SOCKS5，可测试连通性。
- **导出报告**：生成包含扫描目标和删除记录的文本报告。

## 常见问题

### 1. 拖放文件时路径识别失败？
- 请直接使用“添加文件/目录”按钮，或尝试将文件拖放到程序窗口的列表区域（而非标题栏）。

### 2. 在线扫描 GitHub 仓库失败？
- 检查网络连接和代理设置。GitHub 可能对未登录用户有访问限制，但通常公开仓库可正常下载。

### 3. 如何自定义扫描规则？
- 参考 `total_rules.py` 模板，格式如下：
  ```python
  MALICIOUS_PATTERNS = {
      r'os\.system\(.*\)': "调用系统命令",
      r'eval\(.*\)': "动态代码执行",
      # 更多规则...
  }
  ```

### 4. 全盘扫描提示权限错误？
- 某些系统目录（如 `/sys`、`/proc`）可能无法访问，这是正常的，程序会跳过并继续扫描。
